# Slide Selection Guide

## 使い分け

- `cover`
  - 資料の目的、対象、結論の方向を最初に示したいとき
- `problem`
  - 課題、現状、困りごとを導入したいとき
- `solution`
  - 提案内容、フロー、構成、改善方針を説明したいとき
- `comparison`
  - 選択肢、before / after、観点別比較をしたいとき
- `evidence`
  - KPI、実績、グラフ、裏付けを示したいとき
- `decision`
  - scope、trade-off、risk、結論、意思決定条件を整理したいとき
- `action`
  - roadmap、next action、担当と期限を明記したいとき
- `appendix`
  - deep-dive、コード、詳細表、補足図版を本編から分けたいとき

## 代表的な流れ

1. `cover`
2. `problem`
3. `problem` (`current-state`)
4. `solution` (`proposed-solution`)
5. `comparison` or `decision`
6. `solution` (`flow` or `architecture`)
7. `evidence` (`kpi-highlight` / `kpi-detailed`)
8. `decision` (`risk-faq` / `trade-off`)
9. `decision` (`conclusion`)
10. `action` (`next-action`)

## バリアントの考え方

- `comparison-horizontal`
  - 選択肢が 2 つで、左右比較が最も速いとき
- `comparison-vertical`
  - 観点が複数あり、縦積み比較のほうが読みやすいとき
- `kpi-highlight`
  - まず結論に効く数値だけを見せたいとき
- `kpi-detailed`
  - target / actual や注釈を含めて説明したいとき

## 運用ルール

- テーマは見た目だけを変える
- 構造や slide type はテーマで分岐させない
- 役割でページを選び、テーマで見た目を選ぶ
